# OCR 测试脚本命令行化设计

## 目标

将 `test/ocr_test.py` 修复为可直接使用的单图 OCR 命令行工具：规避当前 Windows CPU 环境中的 oneDNN 推理异常，支持从任意工作目录调用，并输出便于阅读的文字与置信度。

## 命令行接口

```powershell
python test/ocr_test.py [图片路径]
```

- 提供图片路径时识别指定图片。
- 未提供路径时识别 `test/raw/Snipaste_2026-06-29_23-06-52.png`。
- 相对路径按调用者当前工作目录解析。
- 图片不存在时向标准错误输出明确原因并以非零状态退出。

## 实现设计

- 使用 `argparse` 解析可选图片参数。
- 默认图片基于 `ocr_test.py` 自身目录定位，消除对当前工作目录的依赖。
- 初始化 `PaddleOCR` 时设置 `enable_mkldnn=False`，规避已复现的 oneDNN `ConvertPirAttribute2RuntimeAttribute` 异常。
- 从 PaddleOCR 结果的 `json["res"]` 中读取 `rec_texts` 和 `rec_scores`。
- 每个文本块输出为 `文本\t置信度`，最后输出识别数量；不输出难以阅读的完整结果对象。
- OCR 初始化和执行保留在 `main()` 中，导入模块时不自动加载模型。

## 错误处理

- 先校验输入路径是已存在文件，避免模型加载后才失败。
- PaddleOCR 自身的推理异常不吞掉，保留堆栈以便定位环境或模型问题。
- 不在本次改动中加入批量目录处理、图像语义理解或自动答题功能。

## 验证

- 单元测试验证默认路径、显式路径、缺失文件和结果文本提取。
- 单元测试不加载 OCR 模型。
- 使用现有 PNG 和 `rag` 环境执行一次完整集成验证，预期退出码为 0，并识别出 49 个文本块。
